Creation d'une clef SSH
> ssh-keygen
Deposer une clef SSH sur UBUNTU
> type $env:USERPROFILE\.ssh\id_rsa.pub | ssh frederic@192.168.0.22 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"


Connaitre la version de ubuntu
> cat /etc/lsb-release

Lister le status du parefeu
CF : https://guide.ubuntu-fr.org/16.04/desktop/net-firewall-on-off.html
> sudo ufw status
Ajouter une regle
> sudo ufw allow 22 (pour SSH par defaut)

Dossier PHP pour Apache2
cd /var/www/html/Echolink

Sur Windows : linux --> win : scp frederic@192.168.0.23:/var/www/html/Echolink/* .
Sur Windows : win --> linux : scp -r * frederic@192.168.0.23:/var/www/html/Echolink/

Pour redémarrer le système Linux à partir d'une session de terminal
sudo reboot

Pour que le systeme redémarre sous GOLANG sans taper le mot de passe 
> whoami
> sudo visudo

Installation de ffmpeg sous ubuntu
> sudo apt install ffmpeg -y

Configuration
> sudo nano /usr/local/etc/svxlink/svxlink.conf

Edition du service
> sudo nano /etc/systemd/system/svxlogd.service
Chargement des services
> sudo systemctl daemon-reload
rendre le service Disponible
> sudo systemctl enable --now svxlogd.service


Lancer le service svxlogd
> sudo systemctl start svxlogd.service # lancement de mon service custom des logs
Lecture des logs
> nano /var/log/svxlink/log_svxlink_2025-11-02.txt
Lancer le service svxlink
> sudo systemctl start svxlink.service # lancement de mon service ECHOLINK
